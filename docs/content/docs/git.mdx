---
title: Git panel
description: A GitKraken-style, read-only git viewer for any session's repo, on any machine — plus lazygit one click away for write operations.
---

Agents change code on machines you are not looking at. The Git panel answers "what did this agent
actually change?" without leaving the app: select a session, press `⇧⌘G`, and the pane becomes a
rich git viewer for **that session's folder, on that session's machine** — a babel node, your Mac,
the Windows box. Toggle again and the terminal is exactly as you left it.

There is nothing to install on any host. The broker answers thin, read-only `/git/*` queries using
the `git` binary every machine already has, and the app renders them in a bundled viewer. No server
runs between clicks, no RAM is held per repo.

## The three views

- **Changes** — the branch, ahead/behind and stash chips, and every staged / unstaged / untracked
  file with status badges and per-file `+/−` counts. It opens straight onto the full working-tree
  diff — GitHub-style side-by-side (or inline), syntax-highlighted, with word-level change
  highlights, per-file collapse, and a filter box.
- **History** — the commit graph. Uncommitted work rides the top as a dashed **WIP row**; lanes are
  smooth curves with merge rings; each row shows ref pills (branch / remote / tag), a color-hashed
  author avatar, hash, and age. Click a commit for its full diff with a file jump-list; **hover** a
  commit to trace its lineage (everything unrelated dims); **⌘-click** a second commit to diff the
  two directly. An **all branches** toggle draws the whole repo's topology in one graph.
- **Blame** — per-line author, commit, and age with a GitLens-style age heatmap; hover for the
  commit summary, double-click a line to jump to that commit in History.

## Agent insights

Agents produce noisy history — many small commits, churny edits, verbose messages. **Shift-click**
a second commit in History to select the whole contiguous range: the panel shows the combined diff
plus an **✦ Agent insights** bar with three levels — **Brief**, **Medium**, **Detailed**. Click one
and a local `claude` run (Sonnet) reads the range — each commit's individual diff *and* the net
effect — and writes what a reviewer would actually want: an upfront account of **what was done**,
then **worth your attention** (risks, leftovers, dead ends, hollow tests), every point anchored to
a file or function.

Cost discipline is built in: nothing generates until you click a level, and results are cached
forever keyed by the commit hashes — commits are immutable, so a cache hit can never go stale and
repeat questions are free. The per-run cost is always shown in the footer.

## Write operations: lazygit

The viewer is deliberately **read-only** — reviewing an agent's work shouldn't risk mutating it.
When you do want to stage, commit, rebase, or resolve conflicts, the panel's **lazygit** button
swaps the same pane to [lazygit](https://github.com/jesseduffield/lazygit), the full-featured
terminal git client, running in that folder on that machine (with mouse support). The broker
installs its single ~6&nbsp;MB binary once per host automatically — on a cluster with a shared home,
one download covers every node.

## How it works

The broker exposes read-only endpoints that shell out to `git` (always with
`--no-optional-locks`, so the viewer can never take an index lock):

| endpoint | purpose |
|---|---|
| `GET /git/summary` | branch, upstream, ahead/behind, stashes, changed files (porcelain v2) |
| `GET /git/log` | commit log with parents + decorations; `all=1` walks every ref, topo-ordered |
| `GET /git/diff` | unified diff — working tree, staged, vs HEAD, one commit, or a commit range |
| `GET /git/blame` | per-line attribution (`blame --porcelain`) |
| `GET /git/show` | a file's content at any revision |
| `POST /gitui` | ensure lazygit + run it in a hidden session for the folder |

The client renders diffs with **diff2html** and the same **highlight.js** the app already bundles,
entirely offline. One viewer instance lives per session and is kept alive across pane switches, so
flipping to a terminal and back doesn't re-render.

<Callout type="info">
Works on every platform the broker runs on — Linux clusters, macOS, and Windows (`git.exe`) — with
zero per-host setup. Only lazygit is fetched, once, and only when you first use the button.
</Callout>

## Opening it

- `⇧⌘G` with a session selected (toggle back the same way)
- The **branch icon** in the pane header
- Right-click a session in the sidebar → **Show Git Panel**
