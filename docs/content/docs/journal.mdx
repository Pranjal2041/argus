When agents do the work, the knowing no longer comes free with the doing. You can supervise twenty
sessions for a week and still struggle to answer “what did I actually get done?” The Activity
Journal is Argus's raw answer: a durable record of the moments **your attention touched the fleet**,
kept so you can inspect them later or derive a higher-level recap.

<Callout type="info">
The canonical journal stays on your Mac: one JSONL file per day in
`~/Library/Application Support/Argus/journal/`, append-only and never expiring. A Settings toggle
turns recording off entirely.
</Callout>

## What gets recorded

The journal is **attention-gated**: raw text is captured at moments you engage; everything else is
small structured references into stores that already exist.

- **Utterances** — typing into a pane is coalesced into messages. Each event records the screen as
  it looked when you began typing, the text actually sent with backspaces applied, and semantic
  keys such as Enter, arrows, Tab, Escape, and Ctrl-C.
- **Outcomes** — a later snapshot of the same pane pairs an utterance with what followed.
- **Silent attention** — watching a session for 20+ seconds without typing records the dwell too.
- **Fleet state and interventions** — Command Center transitions and corrections, Git panel opens
  and insight questions, workflow launches, session create/kill, todos, notes, file saves, W&amp;B
  detections, and similar high-signal actions become small structured events.

## Messages sent from Android

The phone records only messages you actually send from its terminal—no passive dwell, outcome
snapshot, or periodic screen monitoring. It uses the same input parser as macOS, attaches the
screen visible when typing started, and places resolved events in a bounded local outbox.

The phone appends those events to an inbox on the Mac's broker. The Mac drains it with
peek → ingest → acknowledge semantics, preserves the original timestamp, and deduplicates crash
window re-deliveries by event ID. In the ledger, phone-originated messages carry a phone marker.

This is an append log rather than last-write-wins user data: concurrent arrivals cannot overwrite
one another.

## Input privacy and attribution

The journal records utterances as typed. It does **not** guess whether un-echoed terminal input was
a password: the former echo-based rule also discarded legitimate messages when a TUI echoed late
or in an unexpected shape, so it was removed. If your workflow includes typing secrets into a
recorded terminal, turn journal capture off before doing so.

Attribution in the reader is temporal and explicit:

- an utterance screen is labeled **pre-input context**, so text already sitting in a TUI input box
  is not presented as something you sent;
- unsent `❯` suggestions near the bottom of a captured agent screen are tagged as agent
  suggestions when they match no message actually sent that day;
- outcome screens mark the linked message where it appears, separating your words from surrounding
  agent output.

The raw capture remains unchanged; those labels are reader-side interpretation rather than
destructive cleanup.

## The Activity Ledger

Open the in-app ledger with `⇧⌘J` or the book icon. It is a first-class Argus pane that reads the
JSONL files directly—no local HTTP server and no background polling. Choose a day, filter by event
kind or machine, search text, inspect the hour-density strip, and expand an utterance to see its
captured screen.

The original standalone reader remains useful for scripting or an external browser:

```sh
python3 tools/journal_dashboard.py
```

It serves only on `127.0.0.1:8765` and adds a live mode that follows the current file.

## Argus Wrapped

[Argus Wrapped](/docs/wrapped) is the shipped intelligence layer over this record. It computes a
story deck and overview dashboard of your work rhythm, fleet, delegation, interventions, messages,
projects, experiments, and milestones. The statistics are derived locally from the journal rather
than summarized early and thrown away.

Because the raw record remains available, future readers can ask different questions of the same
period without having to predict those questions at capture time.
