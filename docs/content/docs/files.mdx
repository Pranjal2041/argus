Files is a cross-host file browser and editor that rides the **same per-host broker** you already run
everywhere ŌĆö so there is nothing extra to deploy. Open it in its own window and you can browse, edit,
preview, and transfer files on any of your machines, the same way you reach their terminals.

## What it does

- **Browse** any host's filesystem (Mac, Linux, Windows) in a dense, Explorer-style tree.
- **Reveal from a session** ŌĆö jump straight to a session's working directory on its host.
- **Edit** in **Monaco**, the editor that powers VS Code: syntax highlighting for dozens of
  languages, minimap, multi-cursor, find and replace, bracket-pair colors, and the command palette.
- **Per-file tabs** ŌĆö keep several files open at once and switch between them; a dot marks unsaved
  edits, and closing one asks before discarding.
- **Quick-open** ŌĆö press `ŌīśP` to fuzzy-find any file under the current root and jump to it.
- **Go to Folder** ŌĆö press `Ōć¦ŌīśG` and paste or type an absolute path, `~`, `$VAR`, or a
  path relative to the current folder. Completion is live; a directory becomes the tree root and a
  file opens at its parent.
- **Content search** ŌĆö press `Ōć¦ŌīśF` to search text across the open folder, optionally as a regular
  expression. Results are grouped by file with the matching line highlighted; selecting one opens
  it at the exact line.
- **Git-aware tree** ŌĆö modified files are amber, new or untracked files green, and deleted files
  red; parent folders carry a change dot so working-tree activity is visible before opening Git.
- **Markdown preview** ŌĆö for `.md` files, toggle Editor / Split / Preview; the rendered view updates
  live as you type and matches your app theme.
- **Preview anything** ŌĆö images, PDF, audio, and video, with independent zoom.
- **Transfer** ŌĆö upload and download files, with progress.
- **Save a meaningful file to Artifacts** ŌĆö right-click a file and choose **Add to Artifacts**. Files
  reached from a terminal, session sidebar, Git view, or Lab already know their source panel and save
  there in one click. A manually opened Files tab asks which panel should own the snapshot.

## Add a file to a panel's Artifacts

Files are never collected merely because you browse or preview them. When a particular file matters,
right-click it in the tree and choose **Add to _panel_ Artifacts**. Argus stores an immutable local copy
at that moment and adds it to the same simple per-panel library used by Render PDFs and screenshots.
Text, images, and PDFs reopen directly in Argus; other formats use the system preview when available
and can always be exported.

If the file is already open, the snapshot matches the visible version. In particular, an unsaved
Monaco draft is captured without modifying the remote file and is labeled **Draft snapshot**. Files
that are not already loaded are streamed from their host instead of being held in memory. Copies over
100 MB require confirmation and show their size before Argus saves them.

Opening an ordinary file or PDF still does nothing automatically. This action is deliberately explicit,
so low-value browsing never fills the library.

## How it works

The transport is **broker endpoints**, not a separate service or port ŌĆö the same rationale as the
terminal and the port hub. The broker exposes a small `/fs/*` API:

| endpoint | purpose |
|---|---|
| `GET /fs/home` | the host's home directory |
| `GET /fs/list` | resolve and list a directory (entries carry full absolute paths) |
| `GET /fs/find` | recursively list files under a root, for `ŌīśP` quick-open |
| `GET /fs/grep` | bounded content search, using ripgrep when available and a Go walker otherwise |
| `GET /fs/read` | stream a file (`Range` + content-type via `http.ServeContent`) |
| `GET /fs/stat` | resolve a terminal-clicked path on the right host |
| `POST /fs/write` | write a file atomically (temp + rename) |
| `POST /fs/mkdir` | create a directory |
| `POST /fs/rename` | rename / move |
| `POST /fs/delete` | delete |

Path resolution is shared by `/fs/list` and `/fs/stat`, so completion and Enter agree on
relative paths, `~`, `..`, and environment variables even when the broker was launched from a
different working directory.

The editor is Monaco running in a `WKWebView` on macOS, themed from your active app palette so it
matches the rest of the window. Files open editable ŌĆö there is no separate "edit mode" ŌĆö and `ŌīśS`
saves the change back atomically. Markdown preview reuses Argus's offline renderer (Markdown plus
LaTeX and code highlighting), so nothing leaves your machine. Media uses native players, and the
Android client renders PDFs with the platform `PdfRenderer`.

<Callout type="info">
Quick-open's `/fs/find` skips heavy directories (`.git`, `node_modules`, caches, build output) and is
capped in size and time, so it stays fast even on large repos or a remote cluster home.
</Callout>

<Callout type="info">
Content search follows the same bounded-walk discipline. On hosts with `rg` it uses ripgrep; the
broker has a compatible fallback for machines where ripgrep is not installed.
</Callout>

<Callout type="info">
The editor and the folder/interface fonts scale independently, and content panes support
`Ōīś+` / `ŌīśŌłÆ` / `Ōīś0` zoom.
</Callout>
