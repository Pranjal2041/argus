The terminal is the core of Argus. The sidebar lists every session on every machine you can reach —
grouped by machine, then by folder — and opening one streams it live, exactly as if you had run
`tmux attach` on that host, but without the SSH.

## The session list

Sessions are grouped **machine → folder → session**, each row showing a live **state dot** and its
last-activity time, with a count badge per group. The dot is read passively from the screen (no input
is sent to the agent): **running** when the agent's "esc to interrupt" footer is on screen, **idle**
otherwise, and a separate "needs you" signal when it is waiting on input. For the richer, model-written
version of this, see the [Command Center](/docs/command-center).

## Working in a session

- **Live stream** over a binary WebSocket — full input, resize and reflow to your window, a 100k-line
  scrollback, and auto-reconnect with a connection dot in the header.
- **Lifecycle** — create, rename, and kill sessions from the sidebar or the command palette.
- **Find in terminal** (`⌘F`), **scroll to bottom**, and **clear buffer** (`⌘K`).
- **Command palette** (`⌘P`) — jump to any session or run any action, across all machines.
- **Copy / paste / select-all**, and independent font zoom (`⌘+` / `⌘−` / `⌘0`).
- **Click to open** — a file path or URL in the output opens on the right host: a file in
  [Files](/docs/files), a local web address in [Dashboards](/docs/dashboards).

Reconnects are visibility-aware. The pane you are reading uses an eager first-frame watchdog and
retry schedule; background panes back off more calmly so one unhealthy host cannot churn the whole
sidebar. On macOS, the eight most-recent panes stay warm and protocol-current. Off-screen panes keep
parsing output into their existing 100k-line local history, but skip invisible paints; one shared,
bounded main-thread scheduler keeps a burst from several agents from starving clicks or panel
switches. Revealing a pane therefore does not shrink its history or replace it with a reconnect
snapshot. On Windows, a genuinely new connection can initially recover only the recent output held
by the bounded ConPTY ring, but that snapshot preserves DEC private modes such as bracketed paste;
once connected, subsequent output and panel switches retain the full local history normally.

## Hidden panels

Some sessions are noise — a finished job, a scratch shell. **Hide** one (`⇧⌘B` opens the restore
list) and it drops out of the session list and the command center. Hidden state is **owned by the
broker**, so it is remembered across restarts and syncs across your devices: hide a panel on your Mac
and it is hidden on your phone too. Restore it whenever you want.

## Render output

**Render Output** (`⇧⌘M`) lifts the current screen into a static reading view without touching the
live session. **Rendered** is the default: for Codex and Claude panels, Argus recovers the agent's
authored response from its local transcript and uses the visible terminal text to prove that it belongs
to this exact panel. That preserves Markdown headings and lists, GFM tables, links, fenced and
syntax-highlighted code, JSON, and inline or display LaTeX together instead of trying to reconstruct
source syntax that the terminal TUI has already consumed.

The **Terminal** toggle preserves the exact captured grid—ANSI colors, bold and italic text,
underlines, links, spacing, soft wraps, and already-formatted terminal tables. It is also the automatic
fallback for shells, unsupported agents, or any response Argus cannot match with high confidence.
Rendering a selection always uses that selection and is never replaced by transcript content. Source
recovery happens only when you invoke Render Output, and both views run entirely offline, so nothing
leaves your machine.

Clicking **PDF** saves the exact presentation currently on screen—Rendered or Terminal—to Argus's
local **Artifacts** library. Opening Render alone does not save anything, and opening an unrelated
PDF in Files does not add it. Artifacts are grouped by panel and remain available after that panel is
hidden, renamed, stopped, or removed. Open the library with `⇧⌘I`, from the command palette, or from
the archive button in a panel header; there you can search filenames, sort, read, rename, export, or
delete the saved artifacts. The library copy is canonical, while **Export** creates an ordinary copy
wherever you choose. Each explicit save also adds a linked `artifactSaved` entry to the Activity
Ledger.

For quick visual notes, enable **Save foreground clipboard screenshots to Artifacts** in Settings.
When the main Argus window is in front and a panel is visible, a new image placed on the clipboard
(for example with `⌃⇧⌘4`) is copied into that panel's artifact history. Argus stops checking when it
is inactive and resets its clipboard baseline before it resumes, so screenshots taken in other apps
are never imported later. Normal checks read only the pasteboard generation twice per second; image
data is read only after a foreground change, with no keyboard hook, screenshot-folder watcher, or
additional macOS permission.

## Keeping a Mac reachable while locked

On macOS, locking the screen does not pause anything — your sessions, the broker, and the processes
inside them keep running. The only thing that stops them is the Mac going to **sleep**. Turn on **Keep
this Mac awake & reachable while locked** (from `⌘P`, the View menu, or Settings) and Argus holds a
power assertion so the Mac will not idle-sleep: the display still turns off and the screen still locks,
but the system stays awake and your phone keeps reaching every session on it. A coffee-cup indicator
sits in the sidebar header while it is on. It is honored on battery too, and off by default.

<Callout type="warn">
This stops *idle* sleep, not *lid-close* sleep — a closed-lid MacBook on battery still sleeps. Keep the
lid open, or use an external display on power.
</Callout>

<Callout type="info">
Everything here works whether or not anything is attached to the session — the broker reads and
streams from the tmux server (or ConPTY on Windows) directly.
</Callout>

## Connection and UI diagnostics

Argus automatically watches its own main-thread heartbeat. If the app stops servicing the main queue
for two seconds, it captures a five-second process sample under
`~/Library/Logs/Argus/Hangs/` while the blocking stack still exists; it keeps the newest 14 samples.
This is specifically for beachballs that end in Force Quit and therefore never produce a crash
report.

The repository also includes two opt-in flight recorders for broader problems:

- `tools/reconnectwatch.sh` is the repository's cluster diagnostic: it discovers the current
  `ut-babel` / `ut-orchard` peers and probes them with the companion `wsprobe`, requiring real
  frames to arrive. Adapt its discovery filter and tailnet suffix before using it elsewhere.
- `tools/install-hangwatch.sh` installs the macOS `hangwatch.sh` sampler. It records UI latency,
  memory and swap, WindowServer and Argus resource use, and captures heavier process evidence when
  the desktop stalls.

Both write rolling local diagnostics. They are troubleshooting tools, not required services.
