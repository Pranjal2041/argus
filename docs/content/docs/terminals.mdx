The terminal is the core of Argus. The sidebar lists every session on every machine you can reach ŌĆö
grouped by machine, then by folder ŌĆö and opening one streams it live, exactly as if you had run
`tmux attach` on that host, but without the SSH.

## The session list

Sessions are grouped **machine ŌåÆ folder ŌåÆ session**, each row showing a live **state dot** and its
last-activity time, with a count badge per group. The dot is read passively from the screen (no input
is sent to the agent): **running** when the agent's "esc to interrupt" footer is on screen, **idle**
otherwise, and a separate "needs you" signal when it is waiting on input. For the richer, model-written
version of this, see the [Command Center](/docs/command-center).

## Working in a session

- **Live stream** over a binary WebSocket ŌĆö full input, resize and reflow to your window, a 100k-line
  scrollback, and auto-reconnect with a connection dot in the header.
- **Lifecycle** ŌĆö create, rename, and kill sessions from the sidebar or the command palette.
- **Find in terminal** (`ŌīśF`), **scroll to bottom**, and **clear buffer** (`ŌīśK`).
- **Command palette** (`ŌīśP`) ŌĆö jump to any session or run any action, across all machines.
- **Copy / paste / select-all**, and independent font zoom (`Ōīś+` / `ŌīśŌłÆ` / `Ōīś0`).
- **Click to open** ŌĆö a file path or URL in the output opens on the right host: a file in
  [Files](/docs/files), a local web address in [Dashboards](/docs/dashboards).

## Hidden panels

Some sessions are noise ŌĆö a finished job, a scratch shell. **Hide** one (`Ōć¦ŌīśB` opens the restore
list) and it drops out of the session list and the command center. Hidden state is **owned by the
broker**, so it is remembered across restarts and syncs across your devices: hide a panel on your Mac
and it is hidden on your phone too. Restore it whenever you want.

## Render output

Agents often print Markdown, tables, and LaTeX into a terminal, where it reads as raw text. **Render
Output** (`Ōć¦ŌīśM`) lifts the current screen into a clean reading view ŌĆö Markdown, math, and highlighted
code ŌĆö rendered entirely offline, so nothing leaves your machine. It is a reading surface laid over
the terminal, not a replacement for it.

## Keeping a Mac reachable while locked

On macOS, locking the screen does not pause anything ŌĆö your sessions, the broker, and the processes
inside them keep running. The only thing that stops them is the Mac going to **sleep**. Turn on **Keep
this Mac awake & reachable while locked** (from `ŌīśP`, the View menu, or Settings) and Argus holds a
power assertion so the Mac will not idle-sleep: the display still turns off and the screen still locks,
but the system stays awake and your phone keeps reaching every session on it. A coffee-cup indicator
sits in the sidebar header while it is on. It is honored on battery too, and off by default.

<Callout type="warn">
This stops *idle* sleep, not *lid-close* sleep ŌĆö a closed-lid MacBook on battery still sleeps. Keep the
lid open, or use an external display on power.
</Callout>

<Callout type="info">
Everything here works whether or not anything is attached to the session ŌĆö the broker reads and
streams from the tmux server (or ConPTY on Windows) directly.
</Callout>
